---
title: "获取任务详情"
description: "查询异步任务的状态和结果"
api: "GET https://api.ruxa.ai/api/v1/tasks/query/{taskId}"
---

<Info>使用此端点查询通过 API 创建的任何异步任务的状态和结果。</Info>

<Tip>
  生产环境中，建议使用 callback_url
  参数接收生成完成的自动通知，而非轮询状态端点。
</Tip>

## 授权

<ParamField header="Authorization" type="string" required>
  Bearer Token 认证。格式：Bearer sk-xxxxxx
</ParamField>

## 路径参数

<ParamField path="taskId" type="string" required>
  任务 ID，创建任务时返回的唯一标识符
</ParamField>

## 响应

<ResponseField name="code" type="integer" required>
  响应状态码
</ResponseField>

<ResponseField name="message" type="string" required>
  响应消息
</ResponseField>

<ResponseField name="data" type="object" required>
  任务数据对象
</ResponseField>

## 任务状态说明

| 状态       | 描述               |
| ---------- | ------------------ |
| waiting    | 任务已排队等待处理 |
| queuing    | 任务在处理队列中   |
| generating | 任务正在处理中     |
| success    | 任务成功完成       |
| fail       | 任务失败           |

<RequestExample>

```bash cURL
curl --request GET \
  --url https://api.ruxa.ai/api/v1/tasks/query/task_gemini-2.5-flash-image_1766304530229_cf4dafd2 \
  --header 'Authorization: Bearer sk-xxxxxx'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "task_gemini-2.5-flash-image_1766304530229_cf4dafd2",
    "state": "success",
    "resultJson": "{\"resultUrls\":[\"https://cdn.example.com/image.png\"]}"
  }
}
```

</ResponseExample>
